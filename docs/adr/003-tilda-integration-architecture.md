# ADR 003: Интеграция с Tilda и обход LazyLoading

## Статус
Утверждено (Реализовано)

## Контекст (Context)
Требовалось синхронизировать исторические данные по 188 спикерам с legacy-сайта Tilda (`spring.upgrade.st`) в нашу новую БД PostgreSQL. Парсинг напрямую (HTML scraping) был затруднен из-за обфускации кода блоков Tilda и глубокого LazyLoading'а картинок в сетке.

## Решение (Decision)
1. Отказаться от класического скрейпинга `cheerio` на бэкенде.
2. Использовать визуального ИИ-агента (Visual Browser Subagent), который работает через реальный браузер (Playwright/Puppeteer), скроллит страницу, триггерит lazy load, извлекает чистые данные (включая фото) и формирует JSON payloads.
3. Данные инжектятся напрямую в PostgreSQL через защищенный API endpoint `POST /api/speakers/sync-photos` на DevStand.

## Последствия (Consequences)
**Позитивные:** Обход защиты Tilda выполнен успешно; 100% исторических фотографий и подписей синхронизированы без потерь. Был создан стандартный воркфлоу `/tilda_scraping_guide` для будущих миграций.
**Ограничения:** В случае радикального изменения верстки Tilda, визуальному агенту потребуется корректировка селекторов.
