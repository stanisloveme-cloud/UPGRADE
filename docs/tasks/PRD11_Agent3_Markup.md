# TZ: Агент 3 (Subagent, Верстка, Интеграция Tilda)

## Роль и Цель
Ты ИИ-агент, специализирующийся на HTML/CSS верстке, использовании браузерного сабагента (Browser Subagent) для скрапинга и интеграции со сторонними платформами (Tilda). Твоя задача — сверстать "Памятку спикера" (pixel-perfect) и подготовить блок-шаблон для вставки на сайт Tilda.

## Исходные данные
- PRD: `docs/prd/11_Brands_Library_and_Tilda_Integration_RU.md`
- Референс Памятки: `docs/references/Скрины с референсами старой системы.pdf` (или соответствующий скриншот)
- Ссылки на Tilda: `https://spring.upgrade.st/program`, `https://spring.upgrade.st/`

## Задачи

### 1. Верстка "Памятки спикера" (Pixel Perfect)
* Изучи референсы (pdf или картинки). Тебе нужно создать чистую HTML/CSS (или React-компонент, если он рендерится внутри CRM) Памятки.
* Верстка должна быть адаптивной и в точности повторять дизайн со скриншота.
* Документ должен быть готов к печати (print CSS, A4) и корректно отображаться в web-вьювере.

### 2. Скрапинг верстки сетки с сайта Tilda 
* Используй **Browser Subagent**, чтобы зайти на `https://spring.upgrade.st/program`.
* Твоя задача — "стырить" (извлечь) DOM-дерево и CSS стили текущего расписания и карточек спикеров/спонсоров на сайте.
* Сохрани эти данные как шаблон. Нам нужно сгенерировать по ней адаптивный блок `index.html` + `styles.css`.

### 3. Интеграция с Tilda (Zero Block / Custom Code)
* Наш REST API (разрабатывается Агентом 1) будет отдавать JSON по адресу `GET https://devupgrade.space4you.ru/api/public/events/:id/website-data`.
* Напиши ванильный Vanilla JS скрипт, который:
  1. Делает `fetch()` к этому API.
  2. Берет полученный JSON.
  3. Используя шаблон верстки из шага 2, динамически собирает сетку, спикеров и логотипы спонсоров.
  4. Формирует `<div id="crm-schedule-root"></div>`
* Этот код (HTML + CSS + JS) должен быть упакован так, чтобы менеджер мог просто скопировать его, вставить в блок T123 (HTML-код) на сайте Tilda, и всё заработало.
* Убедись, что картинки спикеров, которые подтягиваются из API, на сайте Tilda тоже получают CSS фильтр: `filter: grayscale(100%)`.

### 4. Экстракция старых данных (Опционально)
* Если заказчик предоставит доступ к старой системе в чате, используй Browser Subagent, чтобы авторизоваться там.
* Спарси 20 первых брендов (Название, Описание, Сегменты, Картинки) и сохрани их в JSON файл, чтобы Агент 1 или 2 могли импортировать их в базу.
