# TZ: Агент 2 (Frontend CRM & Спонсоры)

## Роль и Цель
Ты frontend-ориентированный ИИ-агент (React/Vite). Твоя задача — реализовать пользовательский интерфейс для нового раздела "Бренды" и применить визуальные фильтры для спикеров в соответствии с PRD-11.

## Исходные данные
- Интерфейс написан на React с использованием Ant Design.
- Директория фронтенда: `client/`
- PRD: `docs/prd/11_Brands_Library_and_Tilda_Integration_RU.md`

## ВАЖНО: Параллельная работа
Ты будешь работать параллельно с Backend-агентом. Пока реальные эндпоинты не готовы:
- Опирайся на структуры данных, описанные в `prisma/schema.prisma`.
- При необходимости используй mock-данные (заглушки) в сервисах API (например, `client/src/api/sponsors.ts`), чтобы сверстать и оживить интерфейс, а затем переключишь на реальные эндпоинты.

## Задачи

### 1. Глобальный раздел "Бренды"
* **Боковое меню**: Переименуй пункт меню с "Проверка брендов" на "Бренды".
* **Страница списка**: Реализуй таблицу (или грид) для вывода списка всех глобальных брендов. 
* **Модальное окно редактирования бренда**:
  * Измени существующую форму. Удали поля "Телефон финдиректора", "Email финдиректора", "Ссылка на файл".
  * Оставь/Добавь стандартные поля: Название, Описание, Ссылка на сайт, Email, Телефон, Город, Оборот, Штат.
  * **НОВОЕ ПОЛЕ**: "Сегменты рынка". Добавь компонент `Select` в режиме `multiple` для выбора вариантов (придумай или запроси список из 5-10 дефолтных сегментов, например: Retail, IT, Fintech, EdTech, HoReCa, Manufacturing и т.д.).
  * **НОВОЕ ПОЛЕ**: Чекбокс "Выгружать на сайт" (exportToWebsite).
  * **НОВЫЙ ЗАГРУЗЧИК ЛОГОТИПА**: Замени текстовое поле URL на компонент `Upload` (Ant Design). Файл должен отправляться на наш бэкенд (используй или создай роут `/api/uploads/logo`).
    * Добавь валидацию на клиенте: только форматы `.png`, `.svg` и размер не более 10 МБ.

### 2. Привязка Бренда к Мероприятию (Event Settings)
* Зайди на страницу конкретного мероприятия (Schedule/Sponsors tab).
* Нужно переработать логику кнопки "Добавить спонсора". 
* Теперь она должна предлагать два варианта (либо через табы в модалке, либо отдельными кнопками):
  1. **Выбрать бренд**: `Select` с поиском (typeahead) по глобальной базе Брендов, чтобы привязать существующий.
  2. **Создать бренд**: Открывает ту же модалку создания, что и в п.1. После создания он автоматически привязывается к текущему мероприятию.

### 3. Черно-белый фильтр и Чекбоксы спикеров
* В модальном окне редактирования Спикера и Модератора добавь чекбокс "Выгружать на сайт" (exportToWebsite).
* Найди все места в CRM, где отображаются аватарки спикеров (карточки, модалки, сетка). Примени к ним глобальный CSS класс или инлайн-стиль: `filter: grayscale(100%);`. Оригинальное изображение на сервере менять не нужно, только визуальное отображение.
