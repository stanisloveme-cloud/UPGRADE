# **PRD-003: The Grid (Schedule Visualization)**

**Version:** 1.0 **Status:** Ready for Dev **Dependencies:** PRD-002 (Layout ready), PRD-001 (API structure ready)

## **1\. Контекст и Цель**

**Цель:** Реализовать основной интерфейс редактора — визуальную сетку расписания (Timeline), отображающую структуру "Зал → Трек → Сессия". **Сложность:** Высокая. Требует кастомной верстки (CSS Grid/Flex) или использования библиотеки расписаний. **Референс:** Скриншот Редактор деловой программы New Retail Forum 2025\_главная.jpg.

## **2\. Архитектура Компонентов (Grid System)**

### **2.1. Контейнер Сетки (ProgramGrid)**

Компонент, занимающий основную область страницы.

* **Structure:**  
  * **Day Tabs:** Табы переключения дней. Данные генерируются "умным" способом (`Smart Program Dates`): 
    1. Итерируется официальный диапазон Мероприятия (`Event.startDate` -> `Event.endDate`) для генерации гарантированных вкладок (например, 28 и 29 апреля).
    2. К ним добавляются уникальные даты из оставшихся старых или ошибочных треков (Phantom Tabs, например "21 октября"). Как только сессии переносятся и пустой трек удаляется — фантомный таб пропадает.
  * **Time Scale (Header):** Горизонтальная шкала времени.  
  * **Content Area:** Список залов и их наполнение.

### **2.2. Шкала Времени (TimeScale)**

* **Диапазон:** От мин. start\_time до макс. end\_time мероприятия (например, 09:00 — 19:00).  
* **Шаг:** 30 минут.  
* **Рендер:** Метки времени ("09:00", "09:30"...) в верхней части сетки.

### **2.3. Строка Зала (HallRow)**

* **Left Column (Sticky):** Название зала \+ Вместимость (из Hall.name, Hall.capacity).  
* **Track Area:** Горизонтальная область для размещения треков.

### **2.4. Блок Трека (TrackBlock) и TrackModal**

* **Визуализация:** Прямоугольный блок, объединяющий сессии одной тематики.  
* **Header:** Название трека (Track.name) — *Sticky header* внутри блока при скролле (желательно).  
* **Позиционирование:**  
  * Start: Track.start\_time  
  * End: Track.end\_time  
* **Маршрутизация (TrackModal):** Треки привязаны к конкретному `day`. По умолчанию создаваемый трек наследует дату активного таба сетки. Пользователь может изменить дату в выпадающем списке `ProFormSelect` (опции списка — это `availableDays` сетки).
* **Actions:** Кнопка редактирования (карандаш) в заголовке трека.

### **2.5. Карточка Сессии (SessionCard)**

Вложена внутрь TrackBlock.

* **Контент:**  
  * Время: "11:00 — 12:30".  
  * Название: Жирный шрифт.  
  * **Индикаторы (Badges):**  
    * 🟣 (Purple): moderators\_count  
    * 🟢 (Green): speakers\_count  
    * 🔵 (Blue): unconfirmed\_count  
  * **Статус присутствия (Опционально):** Зеленый квадрат, если включен фильтр "Показать статус присутствия".  
* **Interaction:**  
  * onClick: Открывает модалку/drawer редактирования сессии (Функционал Этапа 4).  
  * Hover: Подсветка границ.

## **3\. Логика Рендеринга и CSS**

Для реализации сетки рекомендуется использовать **CSS Grid** или вычисляемое позиционирование (calc()).

**Вариант "CSS Grid" (Рекомендуемый для агента):**

* Ось X (Время): Определить кол-во колонок (Slots) исходя из шага 5-10 минут.  
  * Пример: С 09:00 до 18:00 \= 9 часов \= 540 минут. При шаге 5 мин \= 108 колонок.  
  * grid-template-columns: 200px repeat(108, 1fr); (200px для заголовка зала).  
* Сессия с 11:00 до 12:30:  
  * grid-column-start: Вычисляется от начала дня.  
  * grid-column-end: Вычисляется от длительности.

## **4\. Фильтрация (State Management)**

Подключить чекбоксы из Хедера (PRD-002) к состоянию отображения:

1. hide\_sessions: Если true, скрывать компонент SessionCard, оставлять только TrackBlock (свернутый вид).  
2. hide\_questions: В этом этапе вопросы не рендерятся внутри карточки, но заложить проп.  
3. hide\_speakers: Скрывать бейджи спикеров.  
4. show\_presence\_status: Тогглить видимость индикаторов присутствия.

## **5\. Инструкции для разработчика (Agent Prompt)**

1. **Data Fetching:** Используй хук useRequest (ahooks) или useEffect для получения данных GET /api/events/:id/full-structure.  
2. **Helpers:** Напиши утилиту timeToGridColumn(time, startOfDay), которая преобразует время (строка "11:00") в номер колонки сетки.  
3. **Component Structure:**  
   * Создай components/ScheduleGrid/index.tsx.  
   * Рендери список залов (halls.map).  
   * Внутри зала рендери треки (hall.tracks.map).  
   * Внутри трека рендери сессии (track.sessions.map).  
4. **Styling:**  
   * Зал: Фиксированная высота или min-height (auto-grow).  
   * Трек: Фон bg-gray-100 (или светлый оттенок).  
   * Сессия: Белая карточка с тенью (Card из AntD или div с классами).  
5. **Mock Data:** Обязательно используй богатые моки (много пересекающихся сессий), чтобы протестировать верстку.
