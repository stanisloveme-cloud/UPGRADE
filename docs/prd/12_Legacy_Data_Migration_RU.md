# PRD-12: Миграция исторических данных (Бренды и Спикеры)

## 1. Цель
Наполнить новую глобальную «Библиотеку Брендов» и обновить базу спикеров, мигрировав исторические данные из старой legacy-системы (frontend). Поскольку старая система не поддерживает выгрузку в CSV или обращение к API, миграция будет осуществляться путем автоматизированного парсинга веб-страниц (веб-скрейпинга).

## 2. Контекст
- **Текущее состояние:** Новая CRM-система обладает расширенной схемой для глобальных брендов и спикеров (согласно PRD-11), однако на данный момент базы пусты.
- **Состояние Legacy:** Все исторические данные отображаются исключительно на страницах старого сайта.
- **Расширение контекста:** Старая система является частью сложной связки (CRM Bitrix24 -> Старая система -> Сайт на Tilda). На данный момент устройство интеграции с Bitrix24 неизвестно. Поэтому создание надежного и переиспользуемого инструмента-парсера с интеграцией в новую админ-панель необходимо для потенциального реверс-инжиниринга и задач синхронизации в будущем.

## 3. Объем миграции
1. **Бренды (Спонсоры/Партнеры):**
   - Собрать названия брендов, логотипы (ссылки), краткие описания, веб-ссылки и рыночные сегменты (если доступны).
   - Привязать собранные данные к новой глобальной модели `Sponsor`.
2. **Спикеры (Опционально, но рекомендуется):**
   - Собрать историческую информацию о спикерах (Имя, Компания, Должность, Ссылка на фото, Биография).
   - Предварительно заполнить таблицу `Speaker` для ускорения сборки будущих мероприятий в CRM.

## 4. Технический подход: Кастомный парсер в Admin UI
Мы создадим выделенный Node.js скрипт (например, с использованием Playwright/Puppeteer), интегрированный непосредственно в панель администратора CRM.

### Стратегия: Кастомный скрипт
- Написать отдельный надежный скрипт в бэкенде (например, в пуле `backend/src/tools/scraper`).
- Скрипт авторизуется в старой системе (по предоставленным кредам), проходит по страницам пагинации и извлекает все нужные данные из DOM узлов.
- **КРИТИЧНО:** Скрипт ОБЯЗАН физически скачивать все логотипы и аватары и сохранять их в нашем локальном хранилище (или S3), генерируя новые внутренние URL. Мы не можем полагаться на внешние CDN-ссылки старого сайта.
- Используя `Prisma Client`, напрямую инжектит спарсенные данные в нашу новую базу данных.
- **Интеграция в Admin UI:** Мы выведем механизмы запуска этого скрипта в панель управления CRM. Это позволит администраторам запускать сбор данных по запросу. Это очень важно для будущего реверс-инжиниринга (старая система исторически получала данные из Bitrix24, и этот парсер послужит основой для будущих синхронизаций данных).

## 6. Критерии успеха
- Глобальная библиотека Брендов на 100% заполнена всеми исторически значимыми спонсорами из старой системы.
- Существующие логотипы и веб-ссылки корректно отображаются в новой CRM.
- (Опционально) Справочник спикеров пополнен историческими персонами, готовыми к добавлению в новые сессии.
