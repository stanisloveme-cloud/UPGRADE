# Статус Проекта: UPGRADE CRM (Dev & Prod)

Дата обновления: 17 марта 2026 года

## Текущее состояние инфраструктуры
- **DevStand:** (`devupgrade.space4you.ru`) — Стабилен. Настроен CI/CD для авто-деплоя по пушу в `main`. Версия Nginx корректно принимает переменные окружения для динамического роутинга через `envsubst`.
- **Production:** (`erp-upgrade.ru`) — Стабилен. Пайплайн Github Actions настроен на изолированный запуск по триггеру `workflow_dispatch` на ветке `production`. Схема БД синхронизирована (команда `prisma db push --accept-data-loss` устранила расхождения схемы и API).

## Статус Данных (Миграция Schedule)
Скрипты для обхода проблем с вложенными массивами Prisma и `ON CONFLICT DO UPDATE` (через нативный `pg`) успешно применены:
- **`export_clean.js`**: Экспорт сырых JSON-данных (Спикеры, Треки, Залы) без искажений.
- **`import_prod_data.js`**: Многократно прогнан на сервере, обеспечив "склейку" легаси-данных с новыми связями без нарушения ссылочной целостности и дублей.

## Интеграция с Tilda (Виджет Расписания)
Пиксель-перфектная интеграция завершена и проверена на живых данных Event 76 (`/api/public/events/76/website-data`).

- **Внешний вид:** Воссозданы стили, сетка (`Grid View`), детальный вид сессий (`Detailed View`) полностью идентичны оригинальному лендингу UPGRADE.
- **Маппинг полей:**
  1. Название сессии теперь корректно берется из поля `name` (вместо заглушки "Сессия").
  2. Поле `description` скрыто из детального вида (соответствие с оригинальным дизайном).
  3. Номера вопросов `#1` парсятся из самого текста, убрано ручное дублирование в вёрстке.
  4. **Спикеры и Модераторы:** Значения ролей `moderator` и `Организатор` фильтруются отдельно. Имена модераторов выделяются "жирным" шрифтом (Bold), а должность идет обычным (Regular) шрифтом меньшего размера.
